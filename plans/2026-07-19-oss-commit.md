# OSS 公開用コミット計画

## 目的

ローカル利用者の runtime data、個人情報、端末固有情報、秘密情報を公開対象から除外し、第三者が安全に取得・起動できるソースコードと公開用サンプルだけをコミットする。

## 方針

- `data/scene.sample.json` だけを公開用データとして Git 管理し、それ以外の `data/` 配下はローカル runtime data として除外する。
- runtime data の実体は変更・削除せず、`.gitignore` と Git index で公開対象を制御する。
- コミット前に、現在ファイルと既存 Git 履歴の両方を秘密情報・個人情報・端末固有パスの観点で監査する。
- 既存履歴に公開不可データがある場合、通常の削除コミットだけでは履歴から消えないため、公開前に履歴を除去した新しい公開用履歴を用意する。
- index の全削除状態を解消し、公開対象のみを明示的に stage してから差分を再確認する。

## Acceptance Criteria

- [x] `data/scene.sample.json` 以外の `data/` 配下が `.gitignore` の対象になっている。
- [x] README にローカルデータの扱いとサンプルデータの利用方法が記載されている。
- [x] 現在ファイルを秘密情報候補と端末固有パスについてスキャンした。
- [x] 既存 Git 履歴に公開不可データが残っていないか確認した。
- [x] Git 履歴から過去の runtime data と公開不要な端末固有情報を除去した、または履歴を引き継がない公開用リポジトリを作成した。
- [x] index の意図しない全削除状態を解消し、公開対象だけを stage した。
- [x] stage 後の差分に `data/scene.sample.json` 以外の runtime data、秘密情報、個人情報、端末固有パスが含まれないことを再確認した。
- [x] 型チェックとテストを実行し、成功した。
- [x] OSS ライセンスを決定し、ライセンスファイルを追加した。
- [x] 公開用コミットを作成した。
