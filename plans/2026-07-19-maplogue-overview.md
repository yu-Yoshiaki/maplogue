# Maplogue サービス概要ドキュメント作成計画

## 目的

仮称 Maplogue のコンセプト、対象ユーザー、提供価値、MVP の優先範囲を、公開リポジトリの読者にも伝わる形で整理する。

## 実施内容

1. サービスの核となる「議事録ではなく会議の地図」を言語化する。
2. 原体験、課題、利用シーン、コア体験、既存カテゴリとの差を整理する。
3. 現状と目指す姿、MVP の優先事項と後回しにする事項を明記する。
4. 個人情報や具体的な会議データが含まれていないことを確認する。

## Acceptance Criteria

- [x] 名前と由来（Map + Dialogue）が記載されている。
- [x] 一言説明、背景・原体験、対象ユーザー、ユーザー課題、提供価値が記載されている。
- [x] 典型利用シーンとコア体験が記載されている。
- [x] 議事録 AI、ホワイトボード、マインドマップとの差別化が記載されている。
- [x] 現状と目指す姿、MVP で優先すること、後回しにすることが記載されている。
- [x] 短いタグライン候補が記載されている。
- [x] Maplogue が仮称であり、正式な商標確認前であることが明記されている。
- [x] 個人名、個人メール、ローカル絶対パス、個人の具体的な会議データを含まない。
- [x] Markdown の形式確認、`git diff --check`、禁止情報の検索が完了している。
