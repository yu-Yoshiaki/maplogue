# 公開リポジトリのセットアップ

## 目的

Maplogue を、ローカル依存データや非公開の開発履歴を含まない独立した公開用リポジトリとして準備する。

## 移行元

- 移行元リポジトリ: `realtime-draw`
- 移行元 ref: `codex/oss-initial`
- 移行元コミット: `560bab65b23d9a0ed6dee5f8e5ec6369cb9af621`
- 移行方法: 上記 ref の Git tree のみを新規リポジトリへ展開する

## 安全方針

- 旧 `realtime-draw` リポジトリは非公開の履歴保管用として扱う。
- 新しい `maplogue` リポジトリの `main` を、今後の公開開発の正本とする。
- 旧リポジトリの履歴やブランチを、新リポジトリへ merge しない。
- 公開用サンプルを除く runtime data、個人情報、秘密情報、ローカル絶対パスをコミットしない。
- 公開リモートは、公開前の最終確認後に別途設定する。

## Acceptance Criteria

- [x] 移行元 ref が期待するコミットと一致している。
- [x] 移行元 ref の tree のみから、新規 Git リポジトリが作成されている。
- [x] 新規リポジトリの初期ブランチが `main` である。
- [x] Git remote が設定されていない。
- [x] package name と README の表記が Maplogue に更新されている。
- [x] 型チェック、テスト、ビルド、依存監査が成功する。
- [x] 個人データ、ローカル絶対パス、サンプル以外の runtime data が含まれていない。
- [x] 変更は未コミットのまま Root Orchestrator に引き渡されている。
