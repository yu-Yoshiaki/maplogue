# 整理モード機能 要件入力正本

作成日: 2026-07-10
対象 repo: 本リポジトリ

## ユーザー原文・確定事項

### 整理方針

【Q&A確定】AI 整理モードは有用。通常入力を毎回「TODOモードで」のように説明するのは面倒なので、UI と入力文の両方で切り替えられるようにする。

### UI 操作

【Q&A確定】整理モードの専用ボタンを配置し、手で押して選択できるようにする。

【Q&A確定】ボタンで選んだ整理モードは、次回以降の入力にも使える永続的な選択状態にする。

### スラッシュコマンド

【Q&A確定】音声入力やキーボード入力でも切り替えられるよう、入力文の先頭に `/todo` などを書けば、対応する整理モードボタンがクリックされた状態と同じ扱いにする。

【Q&A確定】スラッシュコマンドは、送信するその入力だけ整理モードを上書きする。ボタンで選択中のモード自体は変更しない。

### 音声入力との関係

【Q&A確定】アプリ内に音声入力機能は持たない。ユーザーは Typeless など OS/外部ツールの音声入力を使う。アプリ側はテキスト入力とスラッシュコマンドを受け取れればよい。

## 採用する仮説

- 初期モードは `normal` とする。
- 最初に提供するモードは `normal` / `todo` / `discussion` / `compare` の4つに絞る。
- スラッシュコマンドは英語短縮形を正とする。日本語別名は実装負荷が低ければ同時に対応してよいが、必須ではない。
- 既存の `data/inbox.jsonl` は後方互換のため、`mode` が無い行を `normal` として扱う。

## 未確定事項

- モード選択状態をブラウザリロード後も保持するか。初期実装では `localStorage` 保存を推奨するが、必須条件ではない。
- 日本語スラッシュ別名（例 `/通常`, `/タスク`, `/議論`, `/比較`）を初期実装に含めるか。
- 監視セッションが `mode` ごとにどこまで既存ノードを書き換えてよいか。初期実装では既存の「最小差分」「明示削除のみ削除」を維持する。

## 除外事項

- アプリ内音声入力。
- 検索・フォーカス。
- 履歴タイムライン。
- ワークスペース切り替え。
- ノード内の大量インライン入力 UI。
