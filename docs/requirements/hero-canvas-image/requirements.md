# ヒーローキャンバス再現 要件定義

作成日: 2026-07-10
対象 repo: 本リポジトリ
入力正本: `docs/requirements/hero-canvas-image/requirements-input.md`

## 目的

`docs/requirements/hero-canvas-image/assets/option-2.png` のヒーローイメージが持つ「書けば、図になる」「手書きの温かみ」「思考がカードや線として育つ」印象を、既存の React Flow キャンバス上で再現する。

画像を単なる背景として貼るのではなく、既存アプリの実キャンバス要素として、カード、付箋、リスト、テーブル、グループ、接続線を表示し、サービスの価値が初見で伝わる状態にする。

## 改修箇所

### 1. キャンバス全体のビジュアルテーマ

- 対象: `src/App.tsx`, `src/styles.css`
- 現状のクリーンな業務ツール風キャンバスを、クリーム系の紙背景、薄いノイズ、やわらかい影、青い手書き風線を持つヒーロー向けテーマに寄せる。
- 通常利用画面を壊さないよう、ヒーロー用テーマは専用 class または専用表示状態で切り替えられる設計にする。

### 2. ノード表現

- 対象: `src/flow/nodes/CardNode.tsx`, `NoteNode.tsx`, `ListNode.tsx`, `TableNode.tsx`, `GroupNode.tsx`, `src/styles.css`
- 参照画像にある「企画メモ」「利用者の声」「次の一手」「優先度」「資料カード群」に相当するノードを、既存ノード種別で表現できるように見た目を調整する。
- カードは白い紙、ノートは付箋、グループは破線の囲み、テーブルは青罫線の表として見えること。

### 3. エッジ表現

- 対象: `src/flow/toFlow.ts`, 必要に応じて custom edge component
- React Flow の標準 smoothstep をそのまま使うだけでなく、青い手書き風の線、接続点、矢印または丸端点を表現できるようにする。
- ヒーロー表示では、カード間の関係が視覚的に追えることを優先する。

### 4. 初期フィットと表示密度

- 対象: `src/App.tsx`, `src/layout/autoLayout.ts`, 必要に応じて pins/sample scene
- ヒーロー用のサンプルシーンが初回表示で画面内に収まり、左のメッセージや下部入力欄と干渉しないようにする。
- 既存の `fitView` と履歴/ワークスペース/入力欄のオーバーレイ位置を考慮する。

## 修正箇所

### 1. 現在のノードデザイン

- 角丸、影、境界線、背景色、余白、文字サイズを、参照画像の温かみと読みやすさに合わせて調整する。
- `note-node` の絵文字依存は、ヒーローでは過度にポップに見える可能性があるため、必要ならアイコン表現や余白を見直す。

### 2. 現在の背景

- `Background gap={24}` のグリッド感がヒーロー画像の紙質感と競合するため、ヒーロー表示では弱める、非表示にする、または紙背景と馴染む色にする。

### 3. 既存 UI オーバーレイ

- `WorkspaceBar`, `scene-title`, `HistoryPanel`, `StatusBadge`, `InputBar` がヒーローの見せ場を隠さないように配置と表示条件を調整する。
- ヒーロー用途では履歴パネルやミニマップが主役を邪魔する場合、初期状態を閉じる、またはヒーロー専用では非表示にする。

### 4. モバイル表示

- 参照画像は横長構図のため、モバイルでは全体縮小だけに頼らず、主要なカードと接続関係が読み取れる表示にする。
- 入力欄や上部バーがカードを覆わないこと。

## 追加箇所

### 1. ヒーロー用サンプルシーン

- `data/hero-scene.json` または既存 scene 管理に沿った専用ワークスペースとして、参照画像に近い内容のデモシーンを追加する。
- 例: 「企画メモ」「利用者の声」「次の一手」「優先度」「資料カード群」を含める。

### 2. ヒーロー用テーマ class

- `hero-canvas` などの class を追加し、通常画面とヒーロー表示でスタイルを分離する。
- 後から通常画面にも適用できるよう、色、影、線、紙質感は CSS 変数化を検討する。

### 3. custom edge

- 手書き風の揺らぎを持つ custom edge を追加する。
- 初期実装では SVG path の stroke linecap/linejoin、破線、軽い影、端点装飾で表現する。

### 4. 視覚回帰確認

- デスクトップとモバイルのスクリーンショット確認を追加する。
- 主要ノード、接続線、入力欄、オーバーレイが重ならないことを確認する。

## 受入れ基準

- [ ] 参照画像の主要要素が、キャンバス上のノード/エッジとして再現されている。
- [ ] ヒーロー表示で「書くと図になる」サービス価値が、静止画ではなく実キャンバスの見た目として伝わる。
- [ ] 既存の通常キャンバス操作、ノードドラッグ、ピン保存、入力送信を壊していない。
- [ ] デスクトップ表示で主要ノードと接続線が初期表示内に収まる。
- [ ] モバイル表示で UI オーバーレイが主要ノードを不自然に隠さない。
- [ ] 参照画像の完全コピーではなく、仕様化された見た目と構造に基づいて再現されている。

## NG 条件

- 参照画像をそのまま背景に貼っただけで、キャンバス要素として操作・表示されていない。
- 通常利用画面の読みやすさや操作性が、ヒーロー用スタイルによって悪化する。
- 履歴、ワークスペース、入力欄など既存 UI がヒーロー表示の主役を隠す。
- ノードや線の装飾が強すぎて、ユーザーの実データが読みづらくなる。
