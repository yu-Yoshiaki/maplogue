# 要件入力

作成日: 2026-07-10
対象 repo: 本リポジトリ
対象画像: `docs/requirements/hero-canvas-image/assets/option-2.png`

## ユーザー原文

> データディレクトリの中に「オプション2」という画像ファイルがあります。これは、このサービスのヒーローイメージになります。
>
> 「こういうことをイメージしているよ」という、このサービスのあり方を示すようなイメージです。画像を見てもらうとわかるのですが、結構手書きの温かみがあるデザインになっています。
>
> これをちょっとキャンバス上で再現したいなと思っているのですが、できそうですか？

> では要件定義を作ってください：
> • 改修箇所
> • 修正箇所
> • 追加箇所

## Q&A確定事項

【Q&A確定】`docs/requirements/hero-canvas-image/assets/option-2.png` は realtime-draw のヒーローイメージであり、サービスのあり方を示す参照画像として扱う。

【Q&A確定】目指す方向性は、画像をそのまま背景として貼ることではなく、キャンバス上の実オブジェクトとして「手書きの温かみがある、書くと図になる体験」を再現すること。

【Q&A確定】既存実装は `@xyflow/react` ベースで、カード、付箋、リスト、テーブル、グループ、接続線をノード/エッジとして表現できるため、既存のキャンバス構造を活かして改修する。

## 未確定事項

- ヒーロー再現キャンバスを、既存アプリの通常画面として常時表示するのか、LP/デモ用の専用モードや専用ワークスペースとして表示するのか。
- 参照画像への忠実度をどこまで求めるか。初期要件では「雰囲気と主要要素の再現」を目標とし、ピクセル単位の完全再現は対象外とする。
- ヒーローキャンバス上の要素をユーザーが編集可能にするか、初期表示サンプルとして読み取り専用に近い扱いにするか。
- 破れた紙の輪郭や紙質感を CSS のみで表現するか、軽量な背景画像/テクスチャ素材を追加するか。
- 既存の通常利用画面にも手書き風テーマを適用するか、ヒーロー用ビューだけに限定するか。

## 除外事項

- 参照画像のピクセル完全コピー。
- SVG や Canvas API で全要素を一から描画する全面作り直し。
- AI 画像生成による新しいヒーロー画像の制作。
- バックエンドの AI 整理ロジック変更。
- 認証、課金、ログイン導線など、参照画像内ナビゲーションの実装。
- 本要件定義からの実装着手。次フェーズへ進むにはユーザー確認を必要とする。
